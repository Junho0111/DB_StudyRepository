### 데이터베이스 관리 (Database Control)

| 기능 | 명령어 (Syntax) | 사용 예시 (Example) |
| :--- | :--- | :--- |
| **생성** | `CREATE DATABASE [DB이름];` | `CREATE DATABASE my_shop;` |
| **사용** | `USE [DB이름];` | `USE my_shop;` |
| **목록 확인** | `SHOW DATABASES;` | `SHOW DATABASES;` |
| **삭제** | `DROP DATABASE [DB이름];` | `DROP DATABASE my_shop;` |

<br>

### 테이블 구조 정의 (Table DDL)

| 기능 | 명령어 (Syntax) | 사용 예시 (Example) |
| :--- | :--- | :--- |
| **생성** | `CREATE TABLE [이름] ( [컬럼] [타입] [제약] );` | `CREATE TABLE sample ( product_id INT PRIMARY KEY, name VARCHAR(100) );` |
| **구조 확인** | `DESC [테이블명];` | `DESC sample;` |
| **목록 확인** | `SHOW TABLES;` | `SHOW TABLES;` |
| **삭제** | `DROP TABLE [테이블명];` | `DROP TABLE sample;` |

<br>

### 데이터 조작 (DML - CRUD)

데이터 관리의 가장 핵심적인 4개 작업입니다. 

#### **Create (데이터 추가)**

* **설명**: 테이블에 새로운 행(Row)을 삽입합니다. 


* **예시**:
```sql
INSERT INTO sample (product_id, name, price) 
VALUES (1, '프리미엄 청바지', 59900); [cite_start]-- 문자열/날짜는 작은따옴표(') 사용 

```

<br>

#### **Read (데이터 조회)**

* **설명**: 저장된 데이터를 불러옵니다. 


* **예시**:
```sql
SELECT * FROM sample; [cite_start]-- 모든 열 조회 
SELECT name, price FROM sample; [cite_start]-- 특정 열만 선택 조회

```

<br>

#### **Update (데이터 수정)**

* **설명**: 기존 데이터를 변경합니다. **WHERE 절이 없으면 모든 데이터가 바뀌므로 주의하세요.** 


* **예시**:
```sql
UPDATE sample 
SET price = 40000 
WHERE product_id = 1; [cite_start]-- id가 1인 행의 가격만 수정 

```

<br>

#### **Delete (데이터 삭제)**

* **설명**: 특정 행을 삭제합니다. 


* **예시**:
```sql
DELETE FROM sample 
WHERE product_id = 1; [cite_start]-- id가 1인 행만 삭제 
```

### 주요 데이터 타입 (Data Types)

* **숫자 (Numeric)** 

* `INT`: 일반적인 정수 (4바이트). 


* `DECIMAL(M, D)`: 정확한 소수점 계산이 필요한 금융 데이터용. 


<br>

* **문자열 (String)** 


* `VARCHAR(n)`: **가변 길이.** 입력된 만큼만 공간 사용 (가장 추천). 


* `CHAR(n)`: **고정 길이.** 주민번호, 국가코드 등 길이가 일정한 데이터용. 


<br>

* **날짜/시간 (Date & Time)** 


* `DATE`: 날짜만 저장 (`YYYY-MM-DD`). 


* `DATETIME`: 날짜+시간 그대로 저장 (2038년 제한 없음, 권장). 


* `TIMESTAMP`: 타임존 자동 변환 (2038년까지만 가능).
